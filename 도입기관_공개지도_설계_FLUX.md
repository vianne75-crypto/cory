# 도입기관 공개 지도·목록 설계

> **대표 인사이트 (2026-06-25)**: 자사몰 신규 거래 발생 시 도입기관 목록에 **자동 등록** → **홈페이지에 지도/목록 공개**.
> **핵심 목적 (2026-06-25 보강)**: **사업장 보건관리자**가 이 목록을 보고 **해당 지역 보건기관에 근로자 보건교육을 의뢰하기 쉽게** 하는 것. 이것이 **보건기관의 알쓰패치 구매 활성화**로 이어질 것이라는 기대.

## 1. 전략 가치 — 수요 창출 매칭 플랫폼

단순 사회적 증거를 넘어, **사업장(수요)과 보건기관(공급)을 연결하는 양면 시장 매칭 도구**.

- **타깃 사용자 = 사업장 보건관리자** — 산안법상 근로자 보건교육 의무 보유. "우리 지역 어느 보건기관이 알쓰패치 교육을 제공하는가"를 지도에서 찾아 **직접 의뢰**
- **보건기관 인센티브** — 관내 사업장 근로자 교육 유치 = 건강증진사업 실적·성과보고. 덕양구 웜미팅 확인: *"기업 의뢰 환영, 기업은 (성과)보고서 쓰기 좋다"*
- **APS 효과** — 사업장 교육 수요 유입 → 보건기관 교육 횟수↑ → **알쓰패치 추가 구매(구매 활성화)**

### 선순환 루프 (growth flywheel)
```
사업장 보건관리자가 도입기관 지도 확인
   → 관내 보건기관에 근로자 보건교육 의뢰
   → 보건기관이 교육 진행 (알쓰패치 사용)
   → 보건기관 알쓰패치 추가 구매 ⟲ (구매 활성화)
   → 도입기관 풀 확대 → 지도 갱신 → 더 많은 사업장 유입
```
→ CLAUDE.md 핵심 코어 **"조직×대상자 이중 루프"** + B2B↔B2G 교차 영업의 구체적 실현체.

### 부수 효과
- **ICP v3.1 영업 톤 시각 증명** — *"같은 광역시 4년 연속 KHEPI 등재"*를 지도로 증명 (사회적 증거)
- **B2B 리드 발생** — 의뢰하는 사업장 = APS 신규 B2B 접점 (관내 기업 적은 보건소 한계 보완)

## 2. 데이터 현황 (cory 실측 2026-06-25)

| 항목 | 수치 |
|------|------|
| 도입 보건소 (purchase_amount>0) | **97개** |
| 도입 보건기관 전체(보건소+전문기관+금연센터+광역) | **198개** |
| 좌표(lat/lng) 보유율 | **96%** → 지도 즉시 가능 |

> ⚠️ **도입 판정 기준은 `purchase_amount>0`(실거래)** 사용. `purchase_stage`는 수기 입력이라 47개로 과소집계 → 부적합. 실거래/주문 이력이 정확.

## 3. "자동 등록"은 이미 80% 구현됨 ✅

```
wcolive 신규 거래 → 애니빌드 → cory orders → X5 스마트 자동매칭(score≥0.7)
   → institutions.purchase_amount 갱신 → 도입기관 풀 자동 편입
```

- 신규 거래 시 별도 작업 없이 도입기관 목록에 자동 반영되는 파이프라인이 **이미 가동 중**(X5 Phase 2 완료분)
- 신규 개발 필요분 = **공개용 데이터 피드 + 지도 UI**뿐

## 4. 공개 데이터 스펙

| 공개 ✅ | 비공개 ❌ |
|--------|----------|
| 기관명·유형·지역(시도/시군구) | 담당자명·연락처 |
| 좌표(지도 핀) | 구매 금액·수량 |
| 도입 연도(최초 거래연도) | 상담 내용·내부 메모 |
| 트랙(선택, 1A/1B/1C) | purchase_stage 등 내부 등급 |

### 공개용 Supabase View 초안 (FLUX 산출)
```sql
CREATE OR REPLACE VIEW public_adopted_institutions AS
SELECT id, name, type, region, district, lat, lng,
       EXTRACT(YEAR FROM first_purchase_date)::int AS adopted_year, track
FROM institutions
WHERE purchase_amount > 0
  AND type IN ('보건소','전문기관','금연지원센터','광역시도 및 중앙기관');
-- GRANT SELECT ON public_adopted_institutions TO anon;  (Supabase Data API 정책)
```
→ 홈페이지(PIXEL)에서 anon key로 fetch → 지도/목록 렌더.

## 5. 공개 채널 + 의뢰 동선 (PIXEL)

- 위치: 자사몰/홈페이지 (보건관리자 다수 방문 — 덕양구 웜미팅서 확인된 동선)
- 형태: ①지역 필터 목록 ②전국 지도(핀/히트맵)
- 인터랙션: 시도→시군구 드릴다운, "우리 지역" 검색

### 🔑 의뢰 동선 (핵심 결정) — 사업장 → 보건기관 연결 방식
| 옵션 | 동작 | 장단 |
|------|------|------|
| A. 보건기관 직접 | 보건소 **공개 대표번호** 노출 → 관리자가 직접 연락 | 단순·신뢰 / APS가 의뢰 흐름 못 봄 |
| B. **APS 중개** ⭐ | "근로자 교육 문의" CTA → APS가 사업장↔보건기관 매칭 | **B2B 리드 확보** + 알쓰패치 사용 보장 + 보건소 영업 명분 |
| C. 하이브리드 | 목록은 공개, 의뢰는 APS 폼 경유 | 권장 — 정보 공개 + 리드 캡처 동시 |

> APS 중개(B/C)가 선순환 루프의 데이터를 닫음 — 의뢰 건수·전환을 cory에서 추적 가능.

## 6. 🔴 검토 필요 (착수 전 필수)

1. **BOND 법적 검토** — ①공공기관(보건소)을 *"특정 상품 도입처"*로 실명 공개 시 공공 중립성·동의 이슈 (대안: 동의 전제 실명 / 카운트·히트맵만 폴백) ②사업장↔보건기관 **근로자 보건교육 의뢰 중개**의 산안법(근로자 보건교육 의무)·의료/교육 위탁 적법성
2. **SAGE 전략** — 노출 범위(전국 vs 지역)·B2G vs B2C 채널·경쟁사 정보 노출 리스크
3. **PIXEL 구현** — 지도 라이브러리·도메인(alth.co.kr 영토)·반응형

## 7. 실행 옵션

- **MVP(빠름)**: 공개 view + 지역별 **목록**만 (실명, 지도 없이) → 1주
- **풀버전**: view + 전국 **지도**(핀·드릴다운) → BOND·PIXEL 협업 2~3주
- **안전형**: 지역별 **도입 카운트 히트맵**(실명 비공개) → 법적 리스크 최소

## 8. 핸드오프 대상 (대표 컨펌 후)
- FLUX: 공개 view 생성 + GRANT + 데이터 피드 (즉시 가능)
- PIXEL: 홈페이지 지도/목록 UI
- BOND: 공공기관 실명 공개 법적 검토
- SAGE: 전략·노출 범위 판단
