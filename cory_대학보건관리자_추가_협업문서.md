# cory CRM — 대학[[Segments/기업_보건관리자|보건관리자]](HC) 고객군 추가 협업문서

> APS고객관리 대시보드(cory)에 "대학보건관리자" 세그먼트를 추가하여
> 2026 HC DM 캠페인 응답부터 본품 구매까지 전체 퍼널을 추적합니다.
> 작성일: 2026-03-08

---

## 1. 배경

| 항목 | 내용 |
|------|------|
| 캠페인 | 2026 HC DM (대학교 건강증진 담당자) |
| DM 발송 | 385건 (2026-03-06 완료) |
| 목표 | QR+전화 합산 8% (31건) |
| 문제 | DM 응답 → 샘플 발송 → 본품 구매 추적이 **cory에 없음** |
| 해결 | cory에 "대학보건관리자" 기관유형 추가 + 퍼널 추적 |

---

## 2. 신규 고객군 정의

### 기존 vs 신규 비교

| 항목 | 전공교육 (기존) | 대학보건관리자 (신규) |
|------|----------------|----------------------|
| 아이콘 | P (분홍) | H (청록 #00897b 제안) |
| 대상 | 교수·학과 | 학생건강센터·[[Segments/대학_보건센터|보건센터]] 담당자 |
| 접점 부서 | 학과 사무실 | 학생처·건강증진센터 |
| 구매 동인 | "수업이 좋아지는가" | "건강증진 프로그램 실적" |
| 구매 주기 | 학기 단위 | 연간 사업계획 ([[Segments/보건소|보건소]]와 유사) |
| 데이터 소스 | `전공교육_구매고객_리스트.csv` (73개교) | `대학교_학생건강센터_주소록.csv` (385개교) |
| 예산 편성 | 학과 예산 | 학생처 예산 (별도 라인) |

> **같은 대학이라도 학과(전공교육)와 건강센터(대학보건관리자)는 별도 의사결정 단위**
> → 중복이 아닌 독립 세그먼트

## 3. cory 반영 사항

### 3-1. 기관유형 추가

```javascript
// js/app.js 또는 filters.js — INSTITUTION_TYPES 에 추가
{
  type: '대학보건관리자',
  color: '#00897b',    // 청록 (teal)
  icon: 'HC',
  label: '대학보건관리자'
}
```

기존 10종 → **11종**

### 3-2. 데이터 import

| 항목 | 값 |
|------|-----|
| 데이터 소스 | `대학교_학생건강센터_주소록.csv` |
| 기관 수 | 385개 |
| 필수 필드 | name, type("대학보건관리자"), region, district, lat, lng |
| 초기 구매단계 | DM 발송 완료 → **"관심"** (385개 전체) |

**import 방법** (택 1):
- A) `data/add_hc_centers.py` 신규 스크립트 작성
- B) 관리자 페이지 → 데이터 업로드 탭 → CSV 일괄 등록
- C) Supabase 직접 import

### 3-3. 구매단계 매핑

DM 캠페인 응답에 따라 자동 전환:

```
DM 발송 완료 (385개)     → 관심 (초기값)
QR 스캔                   → 관심 (유지, 상담내역 기록)
샘플 신청                 → 고려
샘플 수령 + 교육 허브 방문 → 고려 (유지)
본품 구매                 → 구매
재구매                    → 만족
```

### 3-4. 상담내역 연동

| 이벤트 | 상담 태그 | 구매단계 변경 |
|--------|-----------|---------------|
| QR 스캔 | `DM응답`, `QR` | - |
| 전화 문의 | `DM응답`, `전화문의` | - |
| 샘플 신청 | `샘플`, `2026hc` | → 고려 |
| 교육 허브 방문 | `교육허브` | - |
| 본품 구매 | (주문 매칭) | → 구매 |

**데이터 소스:**
- QR 로그: Apps Script 스프레드시트 → 수동 업로드 또는 Webhook 연동
- 전화 문의: 관리자 수동 입력
- 샘플 신청: Apps Script 스프레드시트
- 본품 구매: Order.xls (기존 파이프라인)

## 4. 전공교육(P)과의 관계

### 같은 대학, 다른 세그먼트

```
[서울대학교]
  ├─ 전공교육(P): 보건대학원 보건학과 (교수 대상, 학과 예산)
  └─ 대학보건관리자(HC): 학생건강센터 (담당자 대상, 학생처 예산)
```

### 대시보드에서의 구분

- 필터: 기관유형 "전공교육" vs "대학보건관리자" 개별 선택 가능
- 지도: 같은 대학이라도 **마커 2개** (다른 색상·아이콘)
- 점유율: 별도 분모 (전공교육 73개교 vs 대학보건관리자 385개교)

### 교차 전환 추적

- 전공교육 구매 → 대학보건관리자 소개 (또는 반대) 시 상담 태그로 기록
- 향후: "같은 대학 내 다중 접점" 뷰 추가 가능

### 리포트 사용 = 만족도 지표

```
대학보건관리자가 APS건강행태리포트를 사용

  → 재구매 동인 충족
```
