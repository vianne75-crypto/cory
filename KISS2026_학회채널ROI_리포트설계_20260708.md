## 1. 2026 학회 자체 성과 — 즉시 측정 (7/11)

### 1-A. GA4 퍼널 (client-side · 대표 GA 대시보드에서 export)

측정ID `G-GSL98Z5N92` · 이벤트 흐름:

```
page_view(QR 스캔·랜딩)
  └→ kiss_apply_start (폼 진입)
       └→ kiss_signup (신청 완료) ⭐
  └→ diag_open (진단 열림)
       └→ kiss_diag_submit (진단 제출)
  └→ kiss_flywheel (구매경로 선택: direct/public/inquiry)
```

**핵심 전환율**:
| 지표 | 계산 | 진단 |
|------|------|------|
| 폼 진입율 | `kiss_apply_start / page_view` | 낮으면 = 랜딩 후크 약함 |
| **폼 완료율** | `kiss_signup / kiss_apply_start` | ⭐낮으면 = 폼 마찰(필드·개인정보 동의) |
| 진단 참여율 | `kiss_diag_submit / kiss_signup` | 완료화면 이탈 |
| 핫리드율 | `kiss_flywheel(choice=direct) / kiss_signup` | 직접구매 의향 |

> GA4 → 보고서 → 참여도 → 이벤트, 기간 2026-07-06~10 필터. `page_view`는 위치/QR 소스별(utm) 분해.

### 1-B. cory 서버측 확정 신청 (SQL — 대표 실행 · PII)

```sql
-- 일별 신청·실신청·기관매칭·진단연결
SELECT DATE(created_at AT TIME ZONE 'Asia/Seoul') AS 날짜,
       count(*) AS 신청_전체,
       count(*) FILTER (WHERE ref_key IS NOT NULL) AS 실신청,
       count(*) FILTER (WHERE matched_institution_id IS NOT NULL) AS 기관매칭
FROM kiss_signups
WHERE created_at >= '2026-07-06' GROUP BY 1 ORDER BY 1;

-- 진단·플라이휠 참여
SELECT (SELECT count(*) FROM kiss_diagnosis WHERE ref_key IS NOT NULL) AS 진단_전화연결,
       (SELECT count(*) FROM kiss_diagnosis d WHERE EXISTS
          (SELECT 1 FROM kiss_signups s WHERE s.ref_key=d.ref_key)) AS 진단_신청연결;
SELECT choice AS 구매경로, count(*) FROM kiss_flywheel GROUP BY 1 ORDER BY 2 DESC;

-- 진단 CJM 단계 분포 (청중 인식)
SELECT q4_cjm AS CJM단계, count(*) FROM kiss_diagnosis
WHERE q4_cjm IS NOT NULL GROUP BY 1 ORDER BY 2 DESC;
```

### 1-C. 교차 검증
- GA4 `kiss_signup` 이벤트수 ≈ cory `kiss_signups` 실신청수 (괴리 크면 추적 누락/차단 조사)

## 2. A2 전환 추적 (전향 · D+30 7/10기준 → 8/9, D+90 → 10/8)

2026 KISS 신청자가 실제 **cory 주문**으로 이어지는지:

```sql
-- KISS 신청 → 매칭기관의 신청후 주문 (전환·리드타임)
SELECT s.id AS 신청id, s.institution_name, s.created_at::date AS 신청일,
       o.order_idx, o.reg_time AS 주문일, o.sale_price,
       (o.reg_time::date - s.created_at::date) AS 리드타임_일
FROM kiss_signups s
JOIN institutions i ON i.id = s.matched_institution_id
JOIN orders o ON o.institution_id = i.id
WHERE s.created_at >= '2026-07-06'
  AND o.reg_time::date >= s.created_at::date      -- 신청 이후 주문만
ORDER BY s.created_at;
```
- **전환율** = (주문 발생 신청 기관 수) / (전체 KISS 신청 기관 수)
- **리드타임** = 주문일 − 신청일 중앙값 → 팔로업 타이밍 근거

## 3. A3 활용·재구매 분포 (KISS 유입 도입기관 stage)

```sql
SELECT i.purchase_stage AS 구매단계, i.track AS 트랙, count(DISTINCT i.id) AS 기관수
FROM kiss_signups s JOIN institutions i ON i.id = s.matched_institution_id
WHERE s.created_at >= '2026-07-06'
GROUP BY 1,2 ORDER BY 3 DESC;
```
- 신규(인지/관심) vs 기존(구매/재구매) 비율 → KISS 채널이 **신규 발굴형인지 기존 재활성화형인지** 판정

## 4. A1 반복참석 (베이스라인 — 2027부터 자동)
