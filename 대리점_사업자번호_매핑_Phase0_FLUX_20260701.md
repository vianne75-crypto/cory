## 2. 저장 위치 — Supabase `dealer_business_info` (사외비·anon차단)

`data/dealer-business-info-schema.sql` 테이블. **웹훅·스크래퍼 백필 양쪽이 여기로 upsert**(service_role, mem_id PK). Worker가 로컬파일을 못 써서 테이블로 확정. Phase 2 `issue_invoices_asp.py`는 이 테이블에서 mem_id 조인해 수신자 사업자번호 사용.

> ⚠️ **애니빌드 Pull 조회 API는 없음**(대표 확인 2026-07-01) → 기존 108명은 스크래퍼 1회 백필, **신규는 회원가입 Push Webhook으로 자동**. 재스크래핑 불필요.

## 3. 수집 스크래퍼 (✅ 구조 확정·검증 2026-07-01)

**회원목록 = `/admin/sub_member/mem_list.htm?ajax_yn=0`** (page=N). 사업자번호 헤더 포함 표.

| 데이터 | 위치 |
|--------|------|
| mem_id | `mem_login('XXX')` 함수 인자 (행 innerHTML regex) — 헤더 컬럼엔 없음 |
| 상호 | 셀12 · 회원명 셀13 · 주소 셀14+15 · **사업자번호 셀16** · 이메일 셀17 · 핸드폰 셀18 · **회원등급 셀20** |

**검증 결과**: 대리점 등급(유통회원·APS대리점·대리점) **108명 추출, 사업자번호 99명분**. 목록이 가입일 최신순이라 전 페이지(~300p) 훑어 등급 필터.
- ⚠️ **중복 계정**: 동일 사업자번호에 mem_id 2개(예: kimhs·py0466=소리샘 3170249225) → 발행 시 사업자번호 기준 dedup.
- ⚠️ 사업자번호 누락 9명(바이팜·나누리 등) → 대표 확인 필요.

**최종 스크래퍼 (파일 다운로드 — 콘솔 잘림 없이 전량 확보):**

```javascript
(async function(){
  const U='/admin/sub_member/mem_list.htm?ajax_yn=0';
  const MAXP=300, DEALER=/대리점|유통회원/;
  const found=[]; let er=0, empty=0;
  console.log('=== 대리점 회원 추출 시작 ===');
  for(let p=1;p<=MAXP;p++){
    try{
      const r=await fetch(U+'&page='+p,{credentials:'same-origin'});
      const doc=new DOMParser().parseFromString(await r.text(),'text/html');
      const t=[...doc.querySelectorAll('table')].find(t=>{const hd=t.querySelector('tr');return hd&&/사업자번호/.test(hd.textContent||'');});
      if(!t){if(++empty>=3)break; continue;}
      const rows=t.querySelectorAll('tr');
      if(rows.length<=1){if(++empty>=3)break; continue;}
      empty=0;
      for(let i=1;i<rows.length;i++){
        const tr=rows[i], c=tr.querySelectorAll('td,th');
        const grade=((c[20]||{}).textContent||'').trim();
        if(!DEALER.test(grade))continue;
        const idm=(tr.innerHTML.match(/mem_login\('([^']+)'\)/)||[])[1]||'';
        const g=k=>((c[k]||{}).innerText||(c[k]||{}).textContent||'').trim().replace(/\s+/g,' ');
        found.push({mem_id:idm,grade,company:g(12),name:g(13),business_no:g(16),email:g(17),phone:g(18),addr:(g(14)+' '+g(15)).trim()});
      }
      if(p%20===0)console.log(p+'p, 대리점 '+found.length+'명');
    }catch(e){console.error('p'+p+':'+e.message);if(++er>10)break;await new Promise(z=>setTimeout(z,400));}
    await new Promise(z=>setTimeout(z,80));
  }
  console.log('=== '+found.length+'명 추출 (사업자번호 '+found.filter(x=>x.business_no).length+') ===');
  // dealer_business_info 테이블로 백필 (200건 배치)
  let up=0;
  for(let i=0;i<found.length;i+=200){
    const b=found.slice(i,i+200);
    try{
      const r=await fetch('https://aps-webhook.vianne75.workers.dev/sync-dealer-business',{method:'POST',headers:{'Content-Type':'application/json'},body:JSON.stringify(b)});
      const j=await r.json();
      if(j.success){up+=j.upserted||0;console.log('배치'+(Math.floor(i/200)+1)+': upserted '+j.upserted);}else console.error(j.error);
    }catch(e){console.error(e.message);}
  }
  console.log('=== 완료: dealer_business_info '+up+'명 백필 ===');
})();
```
→ 실행 시 108명이 `dealer_business_info` 테이블로 바로 백필됨. (선행: 테이블 생성 + worker 재배포)

## 4. 🔴 데이터 이슈 — 빈 mem_id 대리점 (6,801,300원·미발행 7)

memlv 3000인데 `mem_id` 빈값 8건. API 발행은 mem_id 조인이 전제라 **이 건은 자동발행 불가**. 
- 원인 후보: 비회원 주문 / mem_id 동기화 누락 / 수기 등록
- 조치: 대표가 해당 주문(order_idx) 확인 → 실제 대리점·사업자번호 수기 확인 → `dealer_business_info`에 order_idx 기반 별도 매핑 or 수동발행(run_workflow.py).

## 4b. 애니빌드 Webhook 연동 (신규·수정 자동)

> 가입시·수정시 정보전송 샘플이 **동일 형식**(json_data, IP화이트리스트, biz_num 포함). 한 엔드포인트로 둘 다 처리.

**Worker 수신**: `POST https://aps-webhook.vianne75.workers.dev/anybuild-member`
- 애니빌드 서버 IP(121.125.73·218.237.67·221.139.49)만 허용, 응답 `OK`
- `json_data` 파싱 → `upsertDealerBusiness` → 대리점 등급(1100/1400/3000)만 `dealer_business_info` upsert(mem_id 병합)
- **가입=INSERT, 수정=UPDATE** 동일 처리

**대표 설정** (애니빌드 API 개발센터):
- **가입시 정보전송** 수신 URL → `.../anybuild-member`
- **회원정보 수정시 정보전송** 수신 URL → `.../anybuild-member` (동일)

→ 신규 대리점·정보 수정 모두 자동 반영. **재스크래핑 불필요**(대표 지속성 우려 완전 해소).
