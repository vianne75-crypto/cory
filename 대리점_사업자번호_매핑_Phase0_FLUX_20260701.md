## 3. 수집 방식 — wcolive 회원정보 스크래퍼 (대표 확인 필요)

주문메모 스크래퍼(`jumun_list.htm`)와 동일 패턴. 단 **회원 관리 페이지 구조가 미확인**이라 대표 1회 진단 필요.

**대표님 — wcolive 관리자 아무 화면 콘솔에서 실행** (회원 목록 URL·사업자번호 위치 탐지):

```javascript
(async function(){
  // 회원목록 후보 경로들 시도 (jumun=sub_sale 패턴 참고)
  const cands=['/admin/member/member_list.htm?ajax_yn=0','/admin/sub_member/member_list.htm?ajax_yn=0','/admin/member/list.htm?ajax_yn=0'];
  for(const u of cands){
    try{
      const r=await fetch(u,{credentials:'same-origin'});
      if(!r.ok){console.log('✗ '+u+' → HTTP '+r.status);continue;}
      const h=await r.text();
      const doc=new DOMParser().parseFromString(h,'text/html');
      const ts=[...doc.querySelectorAll('table')];
      console.log('✓ '+u+' → 테이블 '+ts.length+'개');
      ts.forEach(t=>{const hd=t.querySelector('tr');if(!hd)return;const hs=[...hd.querySelectorAll('th,td')].map(c=>(c.textContent||'').trim().replace(/\s+/g,' ')).filter(Boolean);if(hs.length>=3&&/회원|아이디|사업자|상호|등급/.test(hs.join('')))console.log('   헤더: '+hs.join(' | '));});
      break;
    }catch(e){console.log('✗ '+u+' → '+e.message);}
  }
  console.log('=== 사업자번호가 목록에 없으면 회원 상세페이지에만 있음(→ 상세 순회 방식) ===');
})();
```

→ 출력으로 **회원목록 URL + 사업자번호 컬럼 유무** 확정 후, 스크래퍼 완성본 작성:
- **목록에 사업자번호 있으면** → jumun 스크래퍼처럼 목록 긁기 (mem_id·사업자번호·상호)
- **회원 상세에만 있으면** → 12개 mem_id 상세페이지 순회 (12건이라 부담 없음)

## 4. 🔴 데이터 이슈 — 빈 mem_id 대리점 (6,801,300원·미발행 7)

memlv 3000인데 `mem_id` 빈값 8건. API 발행은 mem_id 조인이 전제라 **이 건은 자동발행 불가**. 
- 원인 후보: 비회원 주문 / mem_id 동기화 누락 / 수기 등록
