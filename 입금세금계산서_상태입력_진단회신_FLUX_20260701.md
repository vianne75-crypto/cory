# 입금·세금계산서 상태 입력 — FLUX 진단 회신 (→ SAGE)

## 핸드오프 (5요소)

- **요약**: SAGE 핸드오프 라이브 진단 완료. `memo_raw` 미채움은 **6월 한정이 아니라 전체 413건 전부 NULL** — 구조적으로 wcolive 관리자 메모가 파이프라인 어디에도 캡처된 적이 없음. 파서·UI는 완비, **상류(수집)만 부재.**

- **핵심 발견 (라이브 실측 2026-07-01)**:
  1. **memo_raw = NULL / 413건 전부** (6월 19건 뿐 아니라 전량). payment_confirmed·invoice_issued·memo_parsed 전부 0건.
  2. **누락 지점 3중 확인** — 메모 필드가 3개 경로 모두에 없음:
     - GAS pull(`Api.gs` action=orders) 응답 키에 memo 없음 (원천 시트에 관리자 메모 컬럼 부재)
     - `order-worker.js` baseRecord(실 Supabase 기록 경로)에 memo_raw 필드 없음 (L253-263)
     - `admin-orders.js` syncOrders 레코드에도 memo_raw 없음 (L242-253)
  3. **구조적 원인**: 입완·전세발행 메모는 주문 **생성 후** 관리자가 입력 → 주문 생성 webhook 페이로드에는 원래 존재하지 않음. wcolive 관리자 주문목록 화면에만 존재.
  4. **파서는 정상** — `admin-memo-parser.js` (입완→payment_confirmed / 전세발행→invoice_issued / MMDD→날짜) 완비. memo_raw만 채워지면 즉시 동작.
  5. **수동 경로 갭**: 입금확인 버튼(`confirmPayment`)은 작동(payment_confirmed=true·발송대기 전환). 그러나 **세금계산서 발행 수동 토글 버튼 없음** — 배지 표시만(L125-126). 지금은 대표가 발행 상태를 수동으로도 못 켬.

- **요청 (SAGE·대표 판단 1건 + FLUX 실행 대기)**:
  - **[선결] 대표 확인**: 입완·전세발행 메모를 지금 **어디에 기입**하시는가? (wcolive 관리자 주문 메모칸 / 별도 시트·수기 / 안 함) → 이 답에 따라 수집 경로(A/B) 결정.
  - **FLUX 실행안** (아래 3경로):
