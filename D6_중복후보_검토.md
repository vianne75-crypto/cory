## 병합 프로토콜 (대표 승인 후 실행)

1. **master 선정**: 위 표의 master ID 사용
2. **데이터 이관**:
   - master.purchase_amount += 병합대상.purchase_amount
   - master.consult_count += 병합대상.consult_count
   - master.metadata 병합 (담당자·연락처 등 보존)
3. **참조 무결성 갱신**:
   - `UPDATE consultations SET institution_id = master_id WHERE institution_id = duplicate_id`
   - `UPDATE orders SET institution_id = master_id WHERE institution_id = duplicate_id`
4. **삭제**: `DELETE FROM institutions WHERE id = duplicate_id`
5. **검증**: 병합 후 master.purchase_amount/consult_count 정합성 확인
