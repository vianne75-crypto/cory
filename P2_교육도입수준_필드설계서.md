# P2 — 교육 프로그램 도입 수준 필드 설계서

> 담당: PULSE 다온 | 작성: 2026-03-17 | 협업: → FLUX 이음 (X2 DB 반영)
> 목적: 기관별 교육 콘텐츠 활용 깊이를 추적하여 재구매율과의 상관관계를 측정

---

## 1. 설계 배경

### 핵심 가설 (검증 중)

```
교육 프로그램 도입 수준이 깊을수록 재구매율이 높다
  패치만 구매 (17% 재구매) → 교육자료 활용 → 시각물 활용 → 성과보고서 활용 (목표 30%+)
```

### 기존 purchase_stage와의 차이

| 구분 | purchase_stage | edu_adoption_level |
|------|---------------|-------------------|
| 의미 | 구매 퍼널 단계 (인지→추천) | 교육 프로그램 도입 깊이 |
| 변화 방향 | 앞으로 진행 (단방향) | 누적 상승 (역행 없음) |
| 측정 대상 | 구매 행동 | 콘텐츠 활용 행동 |
| 목적 | 영업 퍼널 관리 | 재구매 예측 지표 |

→ **별도 필드로 관리** (purchase_stage 대체 X)

---

## 2. 4단계 정의

| Level | 명칭 | 정의 | 해당 고객 특성 |
|-------|------|------|----------------|
| **0** | 미확인 | 도입 수준 미파악 | 신규 또는 데이터 없음 |
| **1** | 패치 단품 | [[Products/알쓰패치|알쓰패치]]만 구매. 교육자료 활용 없음 | 호기심 구매, 1회성 높음 |
| **2** | 교육자료 활용 | 절주 워크시트, 금연 기록지, 강연 시나리오 활용 | 교육 목적 명확, 재구매 가능성 상승 |
| **3** | 시각물 활용 | 캠페인 패널, 부스 배너, 인포그래픽 활용 | 행사·캠페인 진행, 전문성 높음 |
| **4** | 성과보고서 활용 | 사전/사후 설문, 통계 리포트, 사업성과 보고서 활용 | 기관 내 공식 사업, 장기 관계 구축 |

### 단계별 재구매율 추정 (가설, 검증 필요)

| Level | 추정 재구매율 | 근거 |
|-------|-------------|------|
| 1 (패치만) | ~17% | 현재 전체 평균 (검증된 사실) |
| 2 (교육자료) | ~30% | 가설 (S1 BOND 인터뷰로 검증 중) |
| 3 (시각물) | ~45% | 가설 |
| 4 (성과보고서) | ~60%+ | 가설 |

## 3. 기술 설계 — FLUX 이음 전달 사항

### 저장 방식: metadata JSONB (스키마 변경 없음)

```json
{
  "edu_adoption_level": 2,
  "edu_adoption_items": ["절주_워크시트", "강연_시나리오"],
  "edu_adoption_updated": "2026-03-17",
  "edu_adoption_note": "2026 금주 교육 프로그램 활용 확인"
}
```

**선택 이유**: 기존 institutions 테이블 스키마 변경 없이 즉시 반영 가능. ALTER TABLE 불필요.

### 대안: 신규 컬럼 추가 (필요 시)

```sql
ALTER TABLE institutions
  ADD COLUMN edu_adoption_level SMALLINT DEFAULT 0 CHECK (edu_adoption_level BETWEEN 0 AND 4),
  ADD COLUMN edu_adoption_updated DATE;

CREATE INDEX idx_institutions_edu_level ON institutions(edu_adoption_level);
```

**권장**: metadata 방식으로 먼저 운영 후, 쿼리 빈도가 높아지면 컬럼 방식 전환.

## 4. 데이터 입력 방법

### 4-1. 수동 입력 (현재)

cory 대시보드 기관 상세 페이지에서 드롭다운으로 선택:

```
교육 도입 수준: [0. 미확인 ▼]
                 0. 미확인
                 1. 패치 단품
                 2. 교육자료 활용
                 3. 시각물 활용
                 4. 성과보고서 활용
```

### 4-2. 자동 감지 (Phase 2 — FLUX 설계)

| 트리거 | 자동 업데이트 |
|--------|------------|
| 교육자료 첨부 주문 접수 | level → 2 (단, 현재 level < 2 인 경우만) |
| 시각물 패널 주문 접수 | level → 3 |
| 성과보고서 요청 접수 | level → 4 |

## 8. 기존 기관 초기 배치 기준 (일괄 업데이트용)

구매 이력만으로 추정하는 임시 기준 (수동 검토 후 수정):

```sql
-- 임시 초기 배치 (구매 이력 기반 추정)
UPDATE institutions
SET metadata = metadata || jsonb_build_object(
  'edu_adoption_level',
  CASE
    WHEN purchase_stage IN ('추천', '만족') THEN 3   -- 장기 고객 → Lv.3 추정
    WHEN purchase_stage = '구매' AND purchase_volume >= 500 THEN 2  -- 대량 구매 → Lv.2 추정
    WHEN purchase_stage = '구매' THEN 1              -- 일반 구매 → Lv.1
    ELSE 0                                           -- 그 외 → 미확인
  END,
  'edu_adoption_updated', CURRENT_DATE::TEXT,
  'edu_adoption_note', '초기 배치 (구매 이력 기반 추정, 수동 검토 필요)'
)
WHERE metadata->>'edu_adoption_level' IS NULL;
```

> ⚠️ 이 쿼리는 추정값. 실제 교육자료 활용 여부는 개별 확인 필요.
