# 입금·세금계산서 상태 입력 루틴화 (SAGE → PULSE·FLUX)

## 핸드오프 (5요소)

- **요약**: cory `orders`의 입금·세금계산서 상태 필드가 미채워짐(전부 기본값 false/null) → 재무 월결산의 **현금 기준 매출·미수금·세금계산서 발행 추적 불가**. 원인 진단 + 입력 루틴화 요청.
- **핵심 발견** (2026-07-01 라이브 진단):
  1. 인프라는 **이미 완비** — 스키마 필드(`payment_confirmed·payment_date·invoice_issued·invoice_date`)·`js/admin/admin-orders.js` UI·`js/admin/admin-memo-parser.js`(메모 "입완"→입금확인 / "전세(발행)"→세금계산서 자동 파싱).
  2. **근본 원인**: 6월 주문 19건 전부 **`memo_raw` 빈값** → 파서가 돌 데이터가 없음. wcolive 관리자 메모(입금·발행 기록)가 cory로 **동기화되지 않음**. → OPEN 항목 `cory-Webhook출처변경`(페이로드 필드 확장)과 직결.
  3. 결과: 6월 순 수주 16,264,000원 **전액 미입금 표기**(실제 후불 미수 + 데이터 공백 혼재 구분 불가)·세금계산서 발행 0건 표기.
- **요청**:
  - **FLUX**: ① wcolive 주문 **관리자 메모를 webhook/sync 페이로드에 포함** → `memo_raw` 채우기 (Webhook출처변경 연계) ② `admin-memo-parser` **신규 주문 시 정기 실행** → payment/invoice 자동 반영 ③ 대안: `admin-orders` UI에서 **수동 입력 경로** 작동 확인(대표가 직접 입금/발행 체크 가능하게)
  - **PULSE**: 운영 루틴 정례화 — **매주(or 발송·입금 시) 입금·세금계산서 상태 업데이트** 담당. 결과를 [[재무_월간요약_2026]] 현금·미수금 섹션에 피드.
- **긴급도**: 🟡 금주 진단(7월 재무 마감·현금 가시성 목적). ※ F1 파일럿 7/20 판정은 KR1이 주 지표라 본 건은 블로커 아님.
