# 전문기관 광역/권역 관할 확장

> **문제**: 물리적 위치 ≠ 관할 지역. 예) 대전세종금연지원센터는 대전에 있지만 **대전+세종 관할** → 사용자가 지도에서 **세종**을 선택해도 나타나야 함.
> **해결**: `metadata.service_regions`(관할 시도 배열) + `service_scope` 필드 → 지도가 관할 기준으로도 노출.

- **부수 정정**: 광주 광산중독관리통합지원센터(915) region 오류(전남 화순군→광주 광산구) 수정.
- **ICP 근거**: [[Segments/전문기관]] 3유형(중독관리·금연지원센터·정신건강복지센터). 금연지원센터=국가금연지원센터 권역 관할, 광역정신건강복지센터=시도 광역 관할.

## 판정 규칙 (재사용 워크플로)
신규 도입기관 등록 시 아래면 광역/권역 관할:
1. 이름에 **2개+ 시도** 포함 (대전세종·대구경북 등) → service_regions=[해당 시도들], scope=권역
2. 이름에 **"광역"·"권역"** 포함 (광역정신건강복지센터 등) → service_regions=[소속 시도], scope=광역
3. 이름이 **"○○도중독관리센터"** 등 도 단위 → service_regions=[해당 도], scope=광역

## 뷰 반영 (완료 SQL)
`cory/data/name-public-columns.sql` — 뷰에 `service_regions`(jsonb array)·`service_scope` 노출. **대표 Supabase 실행 필요**.

## ⚠️ PIXEL 프론트 필터 로직 (핸드오프)
`웹운영/_dev/adopted-map/index.html` 지역 필터에 관할 매칭 추가:
```js
// 기존: inst.region === selSido && (selGu==='전체' || inst.district===selGu)
// 확장: 관할(service_regions) 포함 시에도 노출
const inRegion = d.region === selSido
  || (Array.isArray(d.service_regions) && d.service_regions.includes(selSido));
// 광역/권역 기관은 해당 시도의 모든 시군구에서 노출 (물리 district 무시)
const inGu = selGu==='전체' || d.district===selGu
  || (d.service_scope && (d.region===selSido || (d.service_regions||[]).includes(selSido)));
const show = inRegion && inGu;
```
→ 예: 사용자가 **세종** 선택 시 대전세종금연지원센터(위치 대전)가 노출. 카드에 "관할: 대전·세종" 배지 권고.
