# 6월 판관비 추출 루트 (SAGE → FLUX)

## 핸드오프 (5요소)

- **요약**: 재무 월결산 **영업이익 마감**에 6월 판관비(인쇄·도구·마케팅)가 필요. 대표가 "cory↔홈페이지 연결로 뽑아올 수 있는 루트가 있다"고 확인 → FLUX가 루트 식별·추출 요청.
- **핵심 발견** (2026-07-01 SAGE 라이브 진단):
  - cory Supabase에 **지출/비용 전용 테이블 없음** (institutions·orders·consultations·settings·dashboard_cache 등만 존재).
  - orders에 `print_type·print_qty·print_date·print_received` 인쇄 필드 있으나 **6월분 미채움**(memo 미동기화와 동일 갭). 현수막·배너는 **고객 주문 line item**(매출측)으로 존재 → 판관비와 직접 매핑 불가.
  - 즉 판관비가 **현재 형태로는 바로 안 뽑힘** → 대표가 아는 홈페이지(wcolive/애니빌드) 연결 경로를 FLUX가 확인해야 함.
- **요청 (FLUX)**:
  1. **추출 루트 식별** — cory↔wcolive(애니빌드) 연결에서 판관비가 뽑히는 실제 경로 확인. 후보: ①애니빌드 관리자 **정산·발주·지출 메뉴** ②인쇄 발주처 연동 데이터(현수막·배너·인쇄물) ③도구 구독·광고비 결제 내역이 흐르는 채널. **대표에게 구체 경로 1회 확인** 권장(대표가 route 인지).
  2. **6월분 추출** — 인쇄(캠페인·학회 인쇄물 발주비) / 도구(구독) / 마케팅(광고·DM) 3분류로 6월 지출 집계.
  3. **재무 시트 반영** — [[재무_월간요약_2026]] §3 6월 블록 "판관비(OpEx)" 채우기 → 영업이익(= 매출총이익 12,417,520 − 판관비) 마감.
  4. (선택) 반복 가능하면 월별 판관비 자동 집계 루트로 정례화(재무 Phase 1 자동화).
- **긴급도**: 🟡 금주 (6월 영업이익 마감용). 수주·매출총이익은 이미 확정(마감 가능), 판관비만 잔여.
